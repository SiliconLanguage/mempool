# MemPool-IPC Phase 1: Native Rust Wrapper

`#![no_std]` bare-metal Rust crate that re-implements the Phase 0 SPSC
lock-free queue using `core::sync::atomic` RVWMO atomics, and provides
safe FFI bindings to the MemPool C runtime (barriers, allocator, MMIO
hardware registers).

Produces a **static library** (`libmempool_ipc_phase1.a`) linked with
MemPool's C boot code (`crt0.S`) via `extern "C" fn main`.

## Inspecting the Binary

### Symbol table

```bash
riscv64-unknown-elf-nm target/riscv32imac-unknown-none-elf/release/libmempool_ipc_phase1.a \
  | grep -E 'T main|U boot_barrier|U ipc_queue|U mempool_'
```

Expected:
```
         U boot_barrier
         U ipc_queue
00000000 T main
```

### Disassembly

```bash
riscv64-unknown-elf-objdump -d \
  target/riscv32imac-unknown-none-elf/release/libmempool_ipc_phase1.a \
  | less
```

Key patterns to verify:
- **Release store:** `fence rw,w` + `sw` (publish path)
- **Acquire load:** `lw` + `fence r,rw` (consume path)
- **SeqCst barrier:** `amoadd.w.aqrl` (hw_barrier)
- **Hardware polling:** `add zero,zero,gp` (NTL.PALL) + `lr.w.aq` + `0x00d00073` (WRS.NTO)

## Struct Layout (ABI Compatibility with Phase 0 C)

The Rust structs use `#[repr(C, align(64))]` to match the C layout exactly:

| Field | C Offset | Rust Offset | Size |
|-------|----------|-------------|------|
| `ring[0].sequence` | `0x00000` | `0x00000` | 4B |
| `ring[0].data` | `0x00004` | `0x00004` | 4B |
| `ring[1].sequence` | `0x00040` | `0x00040` | 4B |
| `next_producer_seq` | `0x10000` | `0x10000` | 64B (aligned) |
| `next_consumer_seq` | `0x10040` | `0x10040` | 64B (aligned) |

The `CachelineU32` wrapper type (`#[repr(C, align(64))]` around `u32`)
replicates C's `__attribute__((aligned(64)))` for struct members.

## Architecture Notes

- **Target ISA:** `riscv32imac-unknown-none-elf` â€” includes the `C`
  (compressed) extension which MemPool hardware does not support. For
  the static library this is acceptable (code size optimization only).
  For final ELF output, pass `-C target-feature=-c` or use a custom
  target JSON to disable compressed instructions.

- **Hardware-assisted polling (Phase 2):** `hardware_spin_wait()` uses
  NTL.PALL (Zihintntl) + LR.W.AQ (A) + WRS.NTO (Zawrs) to suspend the
  hart until the producer invalidates the polled cacheline. Eliminates
  O(NĂ—T) crossbar traffic from naive `lw`+`nop` spin-loops. Requires
  Zawrs + Zihintntl hardware support; NTL.PALL is force-encoded as
  32-bit `.insn r` to prevent C-extension compression.

- **No heap, no OS:** `#![no_std]` + `#![no_main]` + `panic = "abort"`.
  The panic handler loops on `nop` forever.

- **Link-time FFI:** `sync.rs` and `alloc_ffi.rs` declare `extern "C"`
  symbols that resolve when the `.a` is linked with the MemPool runtime
  `.o` files (`synchronization.o`, `alloc.o`). These are not needed for
  the current standalone SPSC test but are available for future phases.
